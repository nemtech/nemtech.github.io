    • サーバ¶

    ブロックチェーンプラットフォームである NEM はスマートアセットトランザクションの強力で、安定した、安全なプラットフォームを提供するために、非中央集権ネットワークのノードに依存しています。

    これらのノードは4年以上稼働し続けてきた、従来 JAVA で書かれていた NEM 分散台帳を C++ でリライトした catapult-server ソフトウェアを使用してデプロイされます。

    その多層アーキテクチャーを通じて、ピアツーピアネットワークの分離と提案された REST API レイヤーを使用して、より優れた API 統合フローを実装することが可能です。

    また Catapult ではプライベートネットワーク向けに catapult-server ノード用の プラグイン を開発することで独自のトランザクションタイプを追加する機能を提供します。

    インストール¶

    Catapult server can be used to configure three different types of nodes:

    • Peer ノード: Peer ノードは API がトランザクションを P2P ネットワークにプッシュすると、それを検証または破棄します。ノードはコンセンサスアルゴリズムを実行して新しいブロックを作成し、そしてネットワークを通して変更を伝播します。
    • API ノード: API ノードの主な役割はトランザクションが検証された後に MongoDB データベースにデータを正しく保存することです。これらはパーシャルなアグリゲートボンドトランザクションを識別して保存もします。
    • Dual ノード: Dual ノードは Peer と API ノードを1つのサーバインスタンスにまとめたものです。

    You can create a new catapult-server image following the building instructions for Ubuntu, but we recommend deploying your own network for development or learning purposes with Catapult Service Bootstrap.

    トラブルシューティング¶

    このセクションでは catapult ノードを設定する際によくあるエラーや問題について説明します。

    モザイクの源泉

    このエラーは通常次のようになります:

    14:19:34.719477 0x000000010dfbf5c0: <error> (local::HostUtils.h@43) Throwing exception: harvesting outflows (0) do not add up to power ten multiple of expected importance (15000000)

    14:19:34.719477 0x000000010dfbf5c0: <error> (local::HostUtils.h@43) Throwing exception: harvesting outflows (15000000) do not add up to power ten multiple of expected importance (17000000)

    何を意味するのか？

    このエラーは以下の状況で発生する可能性があります:

    1. nemesis ブロックの cat.harvest モザイクの供給は config-network.properties ファイル内の供給とは一致しません。
    2. config-network.properties 内の mosaicId は nemesis ブロック内で生成されたものとは一致しません。

    どのように修正するのか？

    最初のシナリオでは、あなたのリソース config-network.properties totalChainImportance を nemesis ブロックで設定された供給に合わせて変更してください。

    2番目のシナリオでは nemgen ツールによって作成された正しいモザイク識別子を見つける必要があります。 nemgen を実行し、結果を txt ファイルに出力することができます。それから cat.<harvest | currency> 識別子を探します。

    パッケージ構成¶

    リポジトリ: catapult-server

    フォルダ名 説明
    /extensions 無垢な catapult-server に機能を追加するモジュール。それらはコンセンサスやネットワーキングのように重要な拡張から ZMQ メッセージングのようなオプションの拡張やその他便利な API まで多岐にわたります。
    /external 使用しているハッシュアルゴリズムの実装。
    /plugins トランザクション経由でチェーンの状態を変更するための新しい方法を紹介するモジュール。
    /resources ノードとネットワークの設定可能プロパティ
    /scripts 開発者向けユーティリティスクリプト
    /sdk テストとツールによる再利用可能なコード
    /seed テスト用ネメシスブロック
    /src Catapult のコアエンジン
    /tests テストの集合
    /tools ネットワークとノードをデプロイとモニタリングするツール

    Tools¶

    Catapult サーバーにはネットワークの健全性チェック、nemesis ブロックの生成、キーペアの作成、突然の終了時のサーバ復旧などを行うツールが付属しています。

    それらは catapult-server/build/bin の中にあります。

    catapult.tools.address¶

    キーペアとアドレスの生成。

    オプション

    -h [ --help ]     print help message
-l [ --loggingConfigurationPath ] arg
                  the path to the logging configuration file
-g [ --generate ] arg (=10)
                  number of random keys to generate
-p [ --public ] arg
                  show address associated with public key
-s [ --secret ] arg
                  show address and public key associated with private key
-n [ --network ] arg (=public)
                  network, possible values: mijin, mijin-test, public (default), public-test
-e [ --useHighEntropySource ]
                  true if a high entropy source should be used for randomness

    コマンド

    ./catapult.tools.address -g  10 -n mijin-test

    出力を保存したい場合は最後に出力ストリームを追加します:

    ./catapult.tools.address -g  10 -n mijin-test > myKeys.txt.

    catapult.tools.nemgen¶

    mijin-test.properties ファイルから nemesis ブロックを生成します。この生成された出力は新しいチェーンの開始に使用されます。

    オプション

    -h [ --help ]     print help message
-l [ --loggingConfigurationPath ] arg
                the path to the logging configuration file
-r [ --resources ] arg (=..)
                the path to the resources directory
-p [ --nemesisProperties ] arg
                the path to the nemesis properties file
-s [ --summary ] arg
                the path to summary output file (default: <bindir>/summary.txt)
-n [ --no-summary ]
                don't generate summary file
-t [ --useTemporaryCacheDatabase ]
                true if a temporary cache database should be created and destroyed

    コマンド

    ./catapult.tools.nemgen -p <nemesis_block_properties_path> -r <resources_path>

    catapult.tools.benchmark¶

    ハッシュや暗号化など Catapult 機能のCPUベースのパフォーマンステストを実行します。

    オプション

    -h [ --help ]     print help message
-l [ --loggingConfigurationPath ] arg
                  the path to the logging configuration file
-t [ --num threads ] arg (=0)
                  the number of threads
-p [ --num partitions ] arg (=0)
                  the number of partitions
-o [ --ops / partition ] arg (=1000)
                  the number of operations per partition
-s [ --data size ] arg (=148)
                  the size of the data to generate

    コマンド

    ./catapult.tools.benchmark

    catapult.tools.network¶

    peers-p2p.json および peers-api.json 内の既知のピアについて、ノード情報、パートナーノード接続情報、およびパートナーノードテーブルを出力します。

    オプション

    -h [ --help ]     print help message
-l [ --loggingConfigurationPath ] arg
                  the path to the logging configuration file
-r [ --resources ] arg (=..)
                  the path to the resources directory

    コマンド

    ./catapult.tools.network -r <resources_path>

    catapult.tools.statusgen¶

    トランザクションの受信後にサーバが返却する可能性のあるステータスエラーのリストを生成します。

    オプション

    -h [ --help ]     print help message
-l [ --loggingConfigurationPath ] arg
                  the path to the logging configuration file

    コマンド

    ./catapult.tools.statusgen

    catapult.tools.health¶

    peers-p2p.json および peers-api.json 内のノードについて、健全性をチェックし、統計情報(未確認のトランザクションキャッシュなど)を表示します。

    オプション

    -h [ --help ]     print help message
-l [ --loggingConfigurationPath ] arg
                  the path to the logging configuration file
-r [ --resources ] arg (=..)
                  the path to the resources directory

    ./catapult.tools.health -r <resources_path>
