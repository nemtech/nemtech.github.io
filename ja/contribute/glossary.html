Glossary¶

絶対総量
モザイクの可分性を考慮したモザイク量を測定する方法。これは相対量に 10divisibility を乗算することで算出します。例えば、モザイクが可分性 2 である場合、10 単位 (相対) を送信するには 1000 (絶対) と定義します。
アカウント
アカウントは秘密鍵によってだけ変更ができるアセットのコンテナです。アカウントには常に 2 つのキー (プライベートとパブリック) とアドレスを持ちます。 詳細 を参照してください。
アカウントリンクトランザクション
A type of transaction used to transfer an account importance score to a proxy account. This is required for all accounts that wish to activate delegated harvesting. Read more.
アカウント制限
ブロックのアナウンスや特定のアカウントへのトランザクションの受信へ対する設定可能なスマートなルールです。 詳細 を参照してください。
アグリゲートトランザクション
ワンタイムの破棄可能なスマートコントラクトを作成することで、複数のトランザクションを一つのトランザクションへマージしたトランザクションタイプです。詳細 を参照してください。
アグリゲートボンド (パーシャルトランザクション)
他の参加者からの署名が必要な場合、アグリゲートトランザクションは ボンデッド になります。 詳細 を参照してください。
アグリゲートコンプリート
要求されている参加者全員が署名をすると、アグリゲートトランザクションはコンプリートになります。 詳細 を参照してください。
エイリアス
The namespace linked to an account or mosaic using Alias transactions. An alias and its linked object can be used interchangeably when sending a transaction. Read more.
API ノード
Nodes responsible for storing data in a readable form in MongoDB. They are also responsible for collecting the cosignatures of aggregated bonded transactions.
デリゲートハーベスターの自動検出
The automatic process by which Symbol servers allows accounts to register as delegated harvesters via special transfer messages.
Catapult
NEM 2.0 の仮の名称 - 最終的に Symbol と命名。
連署名
トランザクションを承認するための署名を提供する行為。
連署者
マルチシグアカウント のアカウントマネージャとして機能するアカウント。連署者はネットワークにアナウンスする前に、マルチシグアカウントトランザクションに署名する必要があります。
クロスチェーンスワップ
第三者 (交換サービスなど) を使用せずに、異なるブロックチェーン間でトークンを取引できる Symbol の組み込み機能。 詳細 を参照してください。
Deadline
A time window for a transaction to be accepted before it reaches its expiration. The transaction is eliminated when the deadline is reached and all the nodes reject the transaction. Read more.
デリゲートハーベスティング
インポータンススコアを新しいプロキシアカウントに委任することにより、ユーザがノードをローカルで実行しなくても報酬を受け取ることができるハーベスト方法。 詳細 を参照してください。
Divisibility
モザイクの小数分割を可能にするプロパティ。可分性はモザイクを分割できる小数点以下の桁数を指定します。
Duration
Length of time measured in ブロック で測定された時間の長さ。Symbol ブロックチェーンの各ブロックはおよそ30秒ごとにハーベストされます。
実行手数料
The fee to be paid for a transaction. Calculated by reading the fee multiplier from the block in which the transaction got confirmed and multiplying it by the size of the transaction.
手数料乗数
A multiplier used to calculate the effective fee of each transaction contained within a block.
グローバル制限
Network-wide rules that determine whether accounts will be able to send or receive a specific mosaic (with Mosaic Restrictions enabled).
ハーベスタ
The account that harvests a block. The account is rewarded with the transaction fees added in the block and the inflation tokens generated.
ハーベスティング
Symbol ブロックチェーンにおいての新しいブロックを作成するプロセスです。 詳細 を参照してください。
ハーベスティング受益者
ノードオペレーターが指定する、ブロック報酬の一部を共有するアカウント。
ハッシュロックトランザクション
A type of transaction which locks funds for a certain amount of blocks. This transaction is required before announcing an Aggregate Bonded Transaction. When the associated Aggregate Transaction is complete, the locked funds are returned to the original account. Read more.
ハッシュタイムロックコントラクト (HTLC)
A protocol which creates a trustless environment for the decentralized exchange of assets. It guarantees that a swap will take place if all the participants agree. On the other hand, if some of them decide not to conclude the process, each participant will receive their locked funds back.
インポータンススコア
A value calculated by the PoS+ protocol based on three factors that determines the probability that an account has to harvest a block.
インフレーション
Network configured increase in currency supply per block. The mosaics created due to inflation are included in the block reward. Read more.
ローカルハーベスティング
ローカルノードで ハーベスティング を実行する方法
最大手数料
トランザクションが承認されるために許可する、送信者によって支払われる、最大 手数料 。
マークル木
A structure of nodes labeled by hashes. It is a data validation technique used by Symbol to store large data associated with a block that cannot be retrieved directly from the block header. It allows light clients to verify if an element (e.g. transaction, receipt statement) exists without demanding the entire ledger history.
メッセージ
トランザクション にテキスト文字列を添付する機能
メタデータ
アカウントやモザイクに添付される追加情報です。 詳細 を参照してください。
最小承認数
Number of cosignatories required for the multisignature account to execute a transaction.
最小削除数
Number of cosignatories required to remove a cosignatory from a multisignature account.
モザイク
固定されたアセット - 変更されない複数ので同一の集合 - を表現するための Symbol ブロックチェーン上のデジタルトークンです。 詳細 を参照してください。
モザイク制限
モザイク作成者にアカウントがアセットを使った取引を制御できるようにする機能。これは作成時に明示的に有効化された制限可能プロパティを持つモザイクにのみ影響します。 詳細 を参照してください。
マルチレベルマルチシグネチャアカウント
An advanced built-in feature of Symbol that allows multisignature accounts to be cosigners for other multisignature accounts, creating multiple layers of cosignatories. Multi-level multisignature accounts add “AND/OR” logic to multi-signature transactions.
マルチシグネチャ (マルチシグ) アカウント
アクション / 転送の開始に (連署者からの) 追加署名が必要なアカウントです。 詳細 を参照してください。
ネームスペース
Symbol ブロックチェーン上の一意なドメインスペース。Symbol アカウントまたはモザイクにリンクできます。インターネットドメインと同様に機能します。 詳細 を参照してください。
Nanowallet
NEM 財団が作成した NIS1 デスクトップウォレット
ネットワーク手数料シンク
An account defined by the network operator that will receive a percentage of the harvesting rewards.
NIS1
基軸通貨 XEM によるパブリックメインネットを動かす NEM ブロックチェーンノードの初期バージョンです。2015年3月31日にローンチされました。
ノード拒否
The act by which the Symbol network will prevent communication with a malicious remote node and reject incoming connections from it.
ノードレピュテーション
A measure of trust that the Symbol network determines for each specific node. The network’s trust for a node increases with each successful interaction, and decreases for each failed attempt of communication.
オフチェーン
Realm outside of the blockchain. Off-chain activity does not directly reflect on the blockchain.
Peer ノード
Nodes that facilitate the blockchain process by verifying transactions and blocks, running the consensus algorithm, creating new blocks, and propagating the changes through the network.
Persistent Delegated Harvesting Unlocking (PDHU)
A feature that enables delegated harvesters to preserve their status despite connectivity problems of nodes. With PDHU, if a node experiences turbulence and reboots, the existing delegated harvesters will automatically reconnect when the node is back online.
秘密鍵
Cryptographic key that gives ultimate control over an account and its assets, and must thus be kept secret. It is paired with the public key in the key pair system.
Proof-of-Stake Plus (PoS+)
Symbol’s consensus protocol. It is a modified PoS protocol which considers users’ activity in the network in addition to their network stakes. The chance that accounts will have to harvest a block is calculated through their importance scores. Read more.
公開鍵
広く播種させることのできるキーペアの公開識別子です。対となる秘密鍵でトランザクションが署名されていることを証明します。公開鍵は秘密鍵から暗号的に導出できます。
レシート
Record of proof for every hidden change on the blockchain. The collection of receipts are hashed into a merkle tree and linked to a block. Read more.
リファレンスモザイク
A mosaic selected by the mosaic creator to define mosaic restrictions that depend directly on the selected mosaic’s global restrictions.
レンタル手数料
Fees required to register a namespace or extend its duration. The default namespace rental fees are configurable per network, but the network dynamically adjusts the namespace rental fees over time.
ロールバック
以前に承認された ブロック を元に戻す行為。
シークレットロックトランザクション
A type of transaction between two accounts where the mosaics remain locked until the recipient presents a valid SecretProofTransaction. Otherwise, the funds are returned to the sender. Read more.
シークレットプルーフトランザクション
A type of transaction through which the recipient of a SecretLockTransaction provides proof and unlocks the mosaics. Read more.
スパムスロットル
大量の未確認のトランザクションでスパムされることに対して Symbol ネットワークの保護を提供する機能。
サブネームスペース
A unique domain on the Symbol blockchain that is a part of a larger domain under the namespace hierarchy. Subnamespaces can only exist in conjunction with a root namespace.
Symbol ソフトウェア開発キット (SDK)
主要なソフトウェア開発ツールは Symbol コンポーネント、このような追加ツール、ライブラリ、アプリケーションなどを作成するためのものです。高度な抽象化により、開発者は特定の API 詳細ではなく製品に集中できます。 詳細 を参照してください。
Symbol コマンドラインインターフェイス (CLI)
Symbol ブロックチェーンと疎通する、オープンソースのコマンドラインインターフェイスです。 詳細 を参照してください。
Symbol 拡張
Plugins that can be added to the Symbol’s protocol to extend its capabilities. Allows developers to introduce different ways to alter the chain’s state via transactions without modifying the core engine or disrupting other features. Read more.
TLS
Symbol ブロックチェーン上でピア間の通信を暗号化するために使用されるセキュリティプロトコル。
トランザクションプラグイン
ネットワークがサポートするトランザクション種類を決定するプラグインのセット。
転送トランザクション
モザイクとメッセージの送信に使用される最も基本的なトランザクション。 詳細 を参照してください。
VRF
VRF stands for verifiable random function. All potential harvesting accounts must link to a second public key by announcing a VrfKeyLinkTransaction. The key linked is then used to randomize block production and leader/participant selection.
XYM
Symbol ブロックチェーンの基軸通貨です。
