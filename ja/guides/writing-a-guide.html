ガイドを記述する¶

新しいガイドの記述を検討してくれてありがとう！

はじめる前に¶

  1. nem2-docs リポジトリ issues を開き、記述しようとしているアイディアを探してください。それらは 「great new issue」 を持っています。
  2. コメントを追加してその1人に貢献するか、あなたのガイドのアイデアに新しいissueを作成するかを選択します。
  3. あなたのガイドが既存のカテゴリに分類できるか確認してください。もしない場合は、新しいissueを作り、カテゴリの提案を行ってください。

  1. このリポジトリのガイドを作成するか、ブログを使用することができます。

このリポジトリへのガイドの記述¶

  1. https://help.github.com/articles/fork-a-repo/ をフォークして`nem2-docs <https://github.com/nemtech/nem2-docs>`_ リポジトリをクローンしてください。
$> git clone <url>
  1. Python 2.7 or 3.4+ と pip がインストールされていることを確認してください。
$> python --version
  1. pipを使用した要求環境のインストール
$> pip install -r requirements.txt
  1. ガイドのフォルダ内に、新しく rst を作成します。
$> mkdir source/guides/<folder_name>/<title>.rst
  1. あなたのコンテンツを整理するためにこのテンプレートに従ってください。
:orphan:

.. post:: 18 Aug, 2018
    :category: monitoring
    :excerpt: 1
    :nocomments:
    :author: <your_name_or_username>

#####
Title
#####

Objective after finishing the guide.

**********
Background
**********

Explain necessary concepts before starting to code.

*************
Prerequisites
*************

- A
- B
- C

************************
Let’s get into some code
************************

Present the code and step-by-step explanation.

************
What's next?
************

Is there any extra exercise that readers could try on their own?
  1. あなたのコードとガイドを書いてください！ 書式は restructured text チートシート を確認してください。
  2. コード例 を``source/resources/examples/<language_or_tool>`` の下に追加してください。rstファイルの中からコード断片をレンダリングすることもできます。
.. example-code::

    .. literalinclude:: <relative_url>.ts
        :language: typescript
        :lines: 20-40

  1. あなたの変更点をテストしプレビューしてください。

    $> make livehtml

  2. 変更をプッシュし、 プルリクエスト をしてください。リポジトリのメンテナがドキュメント記述のスタイルに沿ったコンテンツの構成と編集を行います。

あなたのブログにガイドを書いてください¶

ブログ投稿のリンクを含む既存の問題を作成またはコメントしてください。内容が関連していて繰り返されていない場合、リポジトリの保守担当者はガイドセクションの下にあなたのリンクを含めます。
