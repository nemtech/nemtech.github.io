    • アカウント制限でスパム攻撃を防ぐ¶

    アカウント制限の追加・削除の方法を学ぶ

    背景¶

    あなたの会社は自社製品の品質を証明するためにパブリックチェーンを使用していると想像してください。

    品質検証プロセスが終了すると、オペレータは製品アカウントに 品質シール を送信します。顧客はQRコードをスキャンすることでそれを確認できます。顧客が便利に利用するために、製品アカウントに関連するトランザクションだけを表示して、スパムで煩雑になるのを防ぐ必要があります。

    最終的な顧客はQRコードを読み取って、製品モザイクを検証します。そのため会社としては関連するトランザクションだけを見せたく、それ以外のスパムなどの製品に関係ない情報を避けたいのです。

    スパム攻撃のブロック

    したがって、あなたは一連の条件に従うトランザクションのみを受信するように、製品に アカウント制限 を設定することにしました。

    前提条件¶

    • 転送トランザクション送信ガイド を終了していること
    • モザイクの作成ガイド を終了している

    それではコードを見ていきましょう¶

    ユースケースを解決し始める前に、2つのアカウントを NEM2-CLI で設定する必要があります。

    1. 商品を表すアカウントを作成します。
    nem2-cli account generate

Do you want to save it? [y/n]: y
Introduce NEM 2 Node URL. (Example: http://localhost:3000): http://localhost:3000
Insert profile name (blank means default and it could overwrite the previous profile): product

Public Key:     8DC55282AC40307C230F432EE29E52BD93860C167011B11FA1BAEE124B76AB19
Private Key:    123..456
    1. 会社用に別のアカウントを作成します。
    nem2-cli account generate

Introduce network type (MIJIN_TEST, MIJIN, MAIN_NET, TEST_NET): MIJIN_TEST
Do you want to save it? [y/n]: y
Introduce NEM 2 Node URL. (Example: http://localhost:3000): http://localhost:3000
Insert profile name (blank means default and it could overwrite the previous profile): company

Public Key:     DBA5A88911D01CE951A5DEAFD86108A029EA359BB211B399FC53B8908D6AE272
Private Key:    654..321

    次に特定のモザイクを含む受信転送トランザクションのみを受け入れるように製品アカウントを構成します。

    アドレスによるトランザクションのブロック¶

    アカウントは許可リストの アドレス からのみトランザクションを受信できるようにすることができます。同様にアカウントはトランザクションをブロックしたいアドレスのブロックリストを指定することもできます。

    注釈

    許可とブロック制限は制限タイプごとに相互に排他的です。つまり、アカウントは制限の種類ごとに許可リストまたはブロックリストのいずれかを持つようにしか構成できません。

    デフォルトでは、制限セットがない場合、ネットワーク内のすべてのアカウントが指定されたアカウントにトランザクションをアナウンスできます。

    前の例に戻り、会社アカウントからの受信トランザクションのみを受け入れるように製品アカウントを構成するとします。これを行うには次の手順を実行します。

    1. アカウント制限の変更を定義します。会社アドレス (SBI774-YMFDZI-FPEPC5-4EKRC2-5DKDZJ-H2QVRW-4HBP) を 「許可リスト」 に追加します。
    View Code¶
    const companyAddress = Address.createFromRawAddress('SBI774-YMFDZI-FPEPC5-4EKRC2-5DKDZJ-H2QVRW-4HBP');
const addressRestriction = AccountPropertyModification.createForAddress(PropertyModificationType.Add, companyAddress);
    1. 制限タイプ 「AllowAddress」 の AccountRestrictionTransaction を作成してください。前の手順で作成した変更を配列に追加します。
    View Code¶
    const transaction = AccountPropertyTransaction
    .createAddressPropertyModificationTransaction(
        Deadline.create(),
        PropertyType.AllowAddress,
        [addressRestriction],
        NetworkType.MIJIN_TEST);
    1. 署名してアナウンスします。
    View Code¶
    const productPrivateKey = process.env.PRIVATE_KEY as string;
const networkGenerationHash = process.env.NETWORK_GENERATION_HASH as string;
const productAccount = Account.createFromPrivateKey(productPrivateKey, NetworkType.MIJIN_TEST);
const signedTransaction = productAccount.sign(transaction, networkGenerationHash);

const transactionHttp = new TransactionHttp('http://localhost:3000');
transactionHttp
    .announce(signedTransaction)
    .subscribe(x => console.log(x), err => console.error(err));

    今や 転送トランザクション を送ると、」SBI774-YMFDZI-FPEPC5-4EKRC2-5DKDZJ-H2QVRW-4HBP」 だけが製品へのトランザクション送信が許可されているためエラーが発生します。

    一方であなたの会社アカウントからトランザクションを送った場合は、通常通りに承認メッセージを受け取るでしょう。

    MosaicID によるトランザクションのブロック¶

    会社を表すアカウントが次のモザイクを所有しているとします:

    • company.share: 会社の株式を表現
    • company.quality.seal: 製品の品質テストの合格を表現
    • company.quality.seal: 製品の安全テストの合格を表現

    この場合、製品はシールを受け取ることができるだけで、会社の株は受け取ることができないようにすることに役立ちます。

    したがって、否定を使用することで、製品が会社のアカウントから受け取ることができるトランザクションの種類を絞り込むことができます。シールを明確に許可する代わりに 「company.share」 が含まれる受信トランザクションをブロックするように製品を設定できます。このようにします:

    1. アカウント制限を定義します。」ブロックリスト」 に追加したいモザイクIDを追加してください。
    View Code¶
    const companyShareMosaicId = new MosaicId('2b890153b7a18ff2'); // Replace with the mosaic id representing the company share.
const mosaicRestriction = AccountPropertyModification.createForMosaic(PropertyModificationType.Add, companyShareMosaicId);
    1. 制限タイプ 「BlockMosaic」 の AccountRestrictionTransaction を作成してください。前の手順で作成した変更を配列に追加します。
    View Code¶
    const transaction = AccountPropertyTransaction
    .createMosaicPropertyModificationTransaction(
        Deadline.create(),
        PropertyType.BlockMosaic,
        [mosaicRestriction],
        NetworkType.MIJIN_TEST);
    1. 署名してアナウンスします。
    View Code¶
    const productPrivateKey = process.env.PRIVATE_KEY as string;
const networkGenerationHash = process.env.NETWORK_GENERATION_HASH as string;
const productAccount = Account.createFromPrivateKey(productPrivateKey, NetworkType.MIJIN_TEST);
const signedTransaction = productAccount.sign(transaction, networkGenerationHash);

const transactionHttp = new TransactionHttp('http://localhost:3000');
transactionHttp
    .announce(signedTransaction)
    .subscribe(x => console.log(x), err => console.error(err));

    プロセスが成功すると、製品アカウントは 「company.share」 モザイクが含まれない会社アカウントからのトランザクションだけを受け取ることができます。

    制限の除去¶

    会社が最終顧客に商品を販売した後は、彼らは会社アカウントが商品にトランザクションを送信することのみを許可していたという条件を取り除きたいのです。アカウント制限は設定するのと同じくらい簡単に削除できます。

    1. アカウント制限の変更を定義します。」許可リスト」から会社アドレスを削除します。
    View Code¶
    const companyAddress = Address.createFromRawAddress('SBI774-YMFDZI-FPEPC5-4EKRC2-5DKDZJ-H2QVRW-4HBP');
const addressRestriction = AccountPropertyModification.createForAddress(PropertyModificationType.Remove, companyAddress);
    1. 「AllowAddress」 タイプ を設定して AccountRestrictionTransaction を作成します。作成した変更も追加します。
    View Code¶
    const transaction = AccountPropertyTransaction
    .createAddressPropertyModificationTransaction(
        Deadline.create(),
        PropertyType.AllowAddress,
        [addressRestriction],
        NetworkType.MIJIN_TEST);
    1. 署名してアナウンスします。
    View Code¶
    const productPrivateKey = process.env.PRIVATE_KEY as string;
const networkGenerationHash = process.env.NETWORK_GENERATION_HASH as string;
const productAccount = Account.createFromPrivateKey(productPrivateKey, NetworkType.MIJIN_TEST);
const signedTransaction = productAccount.sign(transaction,networkGenerationHash);

const transactionHttp = new TransactionHttp('http://localhost:3000');
transactionHttp
    .announce(signedTransaction)
    .subscribe(x => console.log(x), err => console.error(err));

    トランザクションが承認された後に、あなたは再び任意のアカウントから製品アカウントにトランザクションを送ることができるはずです。
