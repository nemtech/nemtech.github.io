    • マルチシグアカウントの変更¶

    既存の マルチシグアカウント を変更します。

    まず、1-of-2 マルチシグアカウントを 2-of-2 へ変換します。すると、新しい連署者を追加することで、2-of-3 にできます。その後、連署者を削除し、マルチシグアカウントへ対しての変更操作の方法を理解できるでしょう。

    前提条件¶

    • マルチシグアカウントへの変換ガイド を終了していること
    • マルチシグアカウント1つ
    • アカウントの作成 方法を理解している

    それではコードを見ていきましょう¶

    最小承認の編集¶

    Alice と Bob は 1-of-2 マルチシグアカウントの連署者です。マルチシグトランザクションを承認するには、彼らのうち少なくとも一つのアカウントの署名が必要です。言い換えると 最小承認数 パラメタが現在は 1 に設定されているということになります。

    View Code¶
    const transactionHttp = new TransactionHttp('http://localhost:3000');

const cosignatoryPrivateKey = process.env.COSIGNATORY_1_PRIVATE_KEY as string;
const cosignatoryAccount = Account.createFromPrivateKey(cosignatoryPrivateKey, NetworkType.MIJIN_TEST);

const multisigAccountPublicKey = '202B3861F34F6141E120742A64BC787D6EBC59C9EFB996F4856AA9CBEE11CD31';
const multisigAccount = PublicAccount.createFromPublicKey(multisigAccountPublicKey, NetworkType.MIJIN_TEST);

    マルチシグアカウントはブロックチェーンのレベルで変更可能です。このケースでは、2つの連署者を必要とするように、2-of-2 マルチシグへ移り変わります。これを minApproval パラメタを1つ増加させることによって達成します。

    2-of-2 マルチシグアカウントの例

    アカウントのうち一つ、例えば Alice が、 アグリゲートトランザクション で内包した、minApprovalDelta を増加させる マルチシグアカウント変更トランザクション をアナウンスします。

    1. minApprovalDelta を 1 増やす、マルチシグアカウント変更トランザクションを作ります。
    View Code¶
    const modifyMultisigAccountTransaction = ModifyMultisigAccountTransaction.create(
    Deadline.create(),
    1,
    0,
    [],
    NetworkType.MIJIN_TEST);
    View Code¶
    const modifyMultisigAccountTransaction = ModifyMultisigAccountTransaction.create(
    Deadline.create(),
    1,
    0,
    [],
    NetworkType.MIJIN_TEST);
    1. Wrap the modify multisig account transaction under an aggregate transaction, attaching multisig public key as the signer.

    ネットワークへアナウンスされる前にすべての連署者が署名をすると、アグリゲートトランザクションは コンプリート になります。正当であれば、ブロックへ取り込まれます。

    1つの連署者を必要とするので(1-of-2)、アリスはトランザクションに署名し、それをネットワークへアナウンスできます。

    View Code¶
    const aggregateTransaction = AggregateTransaction.createComplete(
    Deadline.create(),
    [modifyMultisigAccountTransaction.toAggregate(multisigAccount)],
    NetworkType.MIJIN_TEST,
    []);

const signedTransaction = cosignatoryAccount.sign(aggregateTransaction);

transactionHttp
    .announce(signedTransaction)
    .subscribe(x => console.log(x), err => console.error(err));
    View Code¶

    承認されると、マルチシグの最小承認値は 2 となり、2-of-2 マルチシグになります。

    注釈

    minApproval パラメタを減少させ、1-of-2 に戻したい場合は、minApprovalDelta を負の値に設定してください。この場合は -1 です。

    新しい連署者を追加します¶

    2-of-3 連署が必要になるように Alice と Bob は Carol をマルチシグアカウントの連署者に追加したいと思いました。

    2-of-3 マルチシグアカウントの例

    1. マルチシグアカウント変更トランザクション を作成し Carol を連署者として追加します。マルチシグアカウントは 2-of-3 となりますが、最小承認数は増加させていません。
    View Code¶
    const nodeUrl = 'http://localhost:3000';
const transactionHttp = new TransactionHttp(nodeUrl);
const listener = new Listener(nodeUrl);

const cosignatoryPrivateKey = process.env.COSIGNATORY_1_PRIVATE_KEY as string;
const cosignatoryAccount = Account.createFromPrivateKey(cosignatoryPrivateKey, NetworkType.MIJIN_TEST);

const multisigAccountPublicKey = '202B3861F34F6141E120742A64BC787D6EBC59C9EFB996F4856AA9CBEE11CD31';
const multisigAccount = PublicAccount.createFromPublicKey(multisigAccountPublicKey, NetworkType.MIJIN_TEST);

const newCosignatoryPublicKey = 'CD4EE677BD0642C93910CB93214954A9D70FBAAE1FFF1FF530B1FB52389568F1';
const newCosignatoryAccount = PublicAccount.createFromPublicKey(newCosignatoryPublicKey, NetworkType.MIJIN_TEST);

const multisigCosignatoryModification = new MultisigCosignatoryModification(MultisigCosignatoryModificationType.Add, newCosignatoryAccount);

    1. アグリゲートボンドトランザクションを送信する前に、Alice は ロックする のに最低 10 cat.currency を必要とします。このトランザクションはネットワークスパムを防止するためとトランザクションに署名することを促すために要求されます。ハッシュロックトランザクションが承認されると、アグリゲートトランザクションをアナウンスできます。
    View Code¶
    const hashLockTransaction = HashLockTransaction.create(
    Deadline.create(),
    new Mosaic(
        new MosaicId('0dc67fbe1cad29e3'), //Replace with your network currency mosaic id
        UInt64.fromUint(10000000)
    ),
    UInt64.fromUint(480),
    signedTransaction,
    NetworkType.MIJIN_TEST);

const hashLockTransactionSigned = cosignatoryAccount.sign(hashLockTransaction);

listener.open().then(() => {

    transactionHttp
        .announce(hashLockTransactionSigned)
        .subscribe(x => console.log(x), err => console.error(err));

    listener
        .confirmed(cosignatoryAccount.address)
        .pipe(
            filter((transaction) => transaction.transactionInfo !== undefined
                && transaction.transactionInfo.hash === hashLockTransactionSigned.hash),
            mergeMap(ignored => transactionHttp.announceAggregateBonded(signedTransaction))
        )
        .subscribe(announcedAggregateBonded => console.log(announcedAggregateBonded),
            err => console.error(err));
});

    クライアントサイドで使用する場合は(例, Angular, React, Vue) リスナーの実装は変わります 。

    1. Cosign the aggregate transaction hash with Carols’s account. She has to opt-in to become a multisig cosignatory.
    nem2-cli transaction cosign --hash A6A374E66B32A3D5133018EFA9CD6E3169C8EEA339F7CCBE29C47D07086E068C --profile carol
    1. ボブのアカウントで このトランザクションに署名 します。ロックされた cat.currency はまたアリスのアカウントで有効になり、キャロルがマルチシグへ追加されます。
    nem2-cli transaction cosign --hash A6A374E66B32A3D5133018EFA9CD6E3169C8EEA339F7CCBE29C47D07086E068C --profile bob

    連署者を削除します¶

    このガイドを終了したら、マルチシグから連署者を削除してみましょう。全ての連署者を削除することでマルチシグアカウントを通常のアカウントへ変換できます。そのマルチシグの秘密鍵を所有していることを確認してください！

    以下のコードでは 2-of-3マルチシグの連署者を minRemoval を 1 にセットすることで削除する方法です。マルチシグ変更トランザクションはアグリゲートコンプリートによって内包され、他者をマルチシグから削除するのに1人を要求します。

    注釈

    minRemoval パラメタは誰かをマルチシグから削除するために必要な署名数を表します。minApproval パラメタの変更 と同じ方法で増加または減少させることができます。

    View Code¶
    const transactionHttp = new TransactionHttp('http://localhost:3000');

const multisigAccountPublicKey = '202B3861F34F6141E120742A64BC787D6EBC59C9EFB996F4856AA9CBEE11CD31';
const multisigAccount = PublicAccount.createFromPublicKey(multisigAccountPublicKey, NetworkType.MIJIN_TEST);

const cosignatoryToRemovePublicKey = 'CD4EE677BD0642C93910CB93214954A9D70FBAAE1FFF1FF530B1FB52389568F1';
const cosignatoryToRemove = PublicAccount.createFromPublicKey(cosignatoryToRemovePublicKey, NetworkType.MIJIN_TEST);

const cosignatoryPrivateKey = process.env.COSIGNATORY_1_PRIVATE_KEY as string;
const cosignatoryAccount =  Account.createFromPrivateKey(cosignatoryPrivateKey, NetworkType.MIJIN_TEST);

const multisigCosignatoryModification = new MultisigCosignatoryModification(MultisigCosignatoryModificationType.Remove,cosignatoryToRemove);

const modifyMultisigAccountTransaction = ModifyMultisigAccountTransaction.create(
    Deadline.create(),
    0,
    0,
    [multisigCosignatoryModification],
    NetworkType.MIJIN_TEST);

const aggregateTransaction = AggregateTransaction.createComplete(
    Deadline.create(),
    [modifyMultisigAccountTransaction.toAggregate(multisigAccount)],
    NetworkType.MIJIN_TEST,
    []);

const signedTransaction = cosignatoryAccount.sign(aggregateTransaction);

transactionHttp
    .announce(signedTransaction)
    .subscribe(x => console.log(x), err => console.error(err));
