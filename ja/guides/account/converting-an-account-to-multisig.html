マルチシグアカウントへの変換¶

2名の連署者を追加することによって 1-of-2 マルチシグアカウント を作ります。

背景¶

ALICE と BOB は一緒に住んでいて、それぞれの アカウント を持っています。また、BOB が外出するとき、彼が、彼自身と ALICE のために食料雑貨を購入できるように、共有アカウントも持っています。

この共有アカウントは、トランザクションをブロックに取り込んでもらうのに、1人の連署者が必要という意味で、NEM の文脈においては 1-of-2 マルチシグと呼べます。

1-of-2 マルチシグアカウントの例

マルチシグアカウントには連署者アカウントが紐付いていること、そのアカウント自身からはトランザクションを開始できないことを留意してください。連署者だけがトランザクションを開始することができます。

前提条件¶

  • アカウントの作成と開設のガイド を終了していること
  • テキストエディタ または IDE
  • NEM2-SDK または CLI
  • 2つのアカウント(公開鍵)
  • XEM を持ったアカウント1つ

それではコードを見ていきましょう¶

  1. 誰がマルチシグアカウントの連署者になるかを決めます。次に、秘密鍵を提供してマルチシグに変換されるアカウントを開きます。

const transactionHttp = new TransactionHttp('http://localhost:3000');

const privateKey = process.env.PRIVATE_KEY as string; // Private key of the account to convert into multisig
const account = Account.createFromPrivateKey(privateKey, NetworkType.MIJIN_TEST);

const cosignatory1PublicKey = '7D08373CFFE4154E129E04F0827E5F3D6907587E348757B0F87D2F839BF88246';
const cosignatory1 = PublicAccount.createFromPublicKey(cosignatory1PublicKey, NetworkType.MIJIN_TEST);
const cosignatory2PublicKey = 'F82527075248B043994F1CAFD965F3848324C9ABFEC506BC05FBCF5DD7307C9D';
const cosignatory2 = PublicAccount.createFromPublicKey(cosignatory2PublicKey, NetworkType.MIJIN_TEST);

  1. マルチシグアカウント変更トランザクション をセットアップすることでアカウントをマルチシグアカウントへ変換します。1-of-2 マルチシグアカウントとするため、最小署名者は 1 にセットします。
View Code¶
const convertIntoMultisigTransaction = ModifyMultisigAccountTransaction.create(
    Deadline.create(),
    1,
    1,
    [
        new MultisigCosignatoryModification(
            MultisigCosignatoryModificationType.Add,
            cosignatory1,
        ),
        new MultisigCosignatoryModification(
            MultisigCosignatoryModificationType.Add,
            cosignatory2,
        )],
    NetworkType.MIJIN_TEST);
View Code¶
        final ModifyMultisigAccountTransaction convertIntoMultisigTransaction = ModifyMultisigAccountTransaction.create(

            1,
            1,
            Arrays.asList(

                    MultisigCosignatoryModificationType.ADD,

  1. マルチシグアカウント候補のトランザクションへの署名とアナウンス
View Code¶
const signedTransaction = account.sign(convertIntoMultisigTransaction);

transactionHttp
    .announce(signedTransaction)
    .subscribe(x => console.log(x), err => console.error(err));

        final SignedTransaction signedTransaction = account.sign(convertIntoMultisigTransaction);
        transactionHttp.announce(signedTransaction).toFuture().get();

transactionHttp

すべてがうまくいけば、ALICE と BOB はこのマルチシグアカウントの連署者となるでしょう。

注釈

getMultisigAccountInfo 関数を使って、ALICE または BOB が連署者であるマルチシグアカウントの一覧を取得することもできます。

View Code¶
const accountHttp = new AccountHttp('http://localhost:3000');

const address = Address.createFromRawAddress('SCSGBN-HYJD6P-KJHACX-3R2BI3-QUMMOY-QSNW5J-ICLK');

accountHttp
    .getMultisigAccountInfo(address)
    .subscribe(accountInfo => console.log(accountInfo), err => console.error(err));
View Code¶
        final AccountHttp accountHttp = new AccountHttp("http://localhost:3000");
