新しいワークショップの作成¶

新しいワークショップを考えてくれてありがとう！

トピックと対象者の選択から始めます。このリポジトリに公開されているワークショップはオープンソースで、オンラインで利用可能で、書面または記録されたガイドが必要です。

すでに公開されているワークショップをチェックすることを検討してください。最初から完全に新しいワークショップを作成するのではなく、その1人に貢献することができます。

nem2-workshop-template をフォークしてください¶

nem2-workshop-template をフォークすることを推奨します。しかし、前の要件が満たされている場合、提案された構造に従う必要はありません。

  1. Ruby のバージョンが私達が要求しているものに合致するか確かめてください。
$> ruby -v  2.3.3
  1. フォークしたリポジトリをクローンしてください。
$> git clone <url>
  1. gem のインストール
$> bundle install
  1. Jekyll の実行
$> bundle exec jekyll serve

より多くの情報は 公式サイト で確認してください。

_config.yml の編集¶

title: NEM2 Workshop Title
email: your@email.com
description: >- # this means to ignore newlines until "baseurl:"
  Start developing a real use case step by step.
baseurl: "/repository-name" # the subpath of your site, e.g. /blog
url: "https://your-github-username.github.io" # the base hostname & protocol for your site, e.g. http://example.com
repository: your-username/repository-name/

README と index.md を変更してください。¶

目的の学習: ワークショップの対象者と成果を説明してください。

要求事項: ワークショップを実施する人は、開始する前に考慮する必要があることを前提としています。

モジュール: ワークショップの内容を要約してください。

新しいレッスンの追加¶

  1. _lessons の中に新しいマークダウンファイルを作成してください。レッスンファイル名は連番にしてください。

例:

_lessons/01-introduction.markdown
_lessons/02-setting-up-your-workstation.markdown
  1. マークダウンファイルは以下のヘッダーで始まる必要があります。タイトルとパーマリンクの更新を忘れないでください。
---
layout: post
title:  "Introduction"
permalink: /lessons/introduction/
---
  1. 書式は Markdown Cheatsheet を確認してください。
