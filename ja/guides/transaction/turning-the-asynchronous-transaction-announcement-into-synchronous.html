非同期なトランザクションのアナウンスを同期的にする¶

nem2-camel で非同期なトランザクションのアナウンスを同期的にします。

背景¶

ALICE は BOB に 10 nem:xem を送るアプリケーションを開発しています。彼女は BOB にメールを送る前に、トランザクションがネットワークに到着したかどうかを知りたいです。

NIS1 では、トランザクションをアナウンスしたら、ノードからのレスポンスを待つ必要がありました。Catapult では異なる動作をします。トランザクションがアナウンスされると、REST API サーバーは常に OK を返します。

これにより、開発者はサーバがレスポンスを返却するのを待つ必要がなくなり、よりレスポンシブなアプリケーションの開発を可能になりました。しかし、トランザクションの状態を確認、承認を確認することは開発者の責任となりました。

一方で、トランザクションの状態を追跡し続けることは、小さなプロジェクトに複雑さを持ち込むことになります。NIS1 からのマイグレーションにおいても難易度が上がってしまうでしょう。

nem2-camel aims to solve these problems by providing a server that listens to the Catapult REST calls and acts as a proxy. When it detects a transaction announcement, it waits for the confirmation via WebSockets and returns the message to the HTTP call.

注釈

nem2-camel をプロキシとして使用するとき、 TransactionHttp.announceSync 関数はトランザクションを同期的に実行します。nem2-camel は、トランザクションがネットワークへ到着し、正当性チェックエラーが無い場合に、成功を返します。あなたは、追加のアクションを実行する前に 幾分かの承認を待つ 必要があるかもしれません。

nem2-camel

前提条件¶

  • 転送トランザクションの送信ガイド を終了していること
  • テキストエディタ または IDE
  • XEM を保有したアカウント

それではコードを見ていきましょう¶

Catapultサービスをローカルに立ち上げます

nem2-camel はアプリケーションと REST API の間のプロキシとして動作します。

開発と学習の目的のために Catapult Service Bootstrap を使用して、 Catapult サーバ と Catapult REST を実行できます。

  1. 以下のインストラクションを実行する前に、 docker と docker compose をインストールしてください:
$> git clone git@github.com:tech-bureau/catapult-service-bootstrap.git
$> cd catapult-service-bootstrap
$> docker-compose up
  1. すべてがうまく行けば、イメージがダウンロードされ、サービスが動き始めるので、最初のブロックの情報を取得して確認ができるでしょう:
$> curl localhost:3000/block/1

ALICE と BOB のアドレスを手に入れる

Catapult サービスが動き出せば、XEM を保有した アカウント が生成されるでしょう。

  1. nemesis_addresses のセクションからキーペアを見つけてください。
   $> cd  build/generated-addresses/
   $> cat raw-addresses.yaml

2. Take the first key pair as Alice's account, and copy the private key.

2. Take the second key pair as Bob's account, and copy the address.

nem2-camel のインストール

注釈

nem2-camel は少なくとも Java 8 以上を必要とします。

  1. リリースされている nem2-camel jar パッケージをダウンロードして、実行してください:
$> java -jar nem2-camel.jar --url http://localhost:3000
  1. サービスを起動したら 0.0.0.0:9000 を新しいプロキシURLとして使用してください。

転送トランザクションの送信

  1. ALICE は 転送トランザクション を作成し、10 XEM を BOB へ送り、彼女のアカウントで署名します。
View Code¶
const privateKey = process.env.PRIVATE_KEY as string;
const account = Account.createFromPrivateKey(privateKey,NetworkType.MIJIN_TEST);

const recipientAddress =  Address.createFromRawAddress('SBHEVGUFDEW22FAT2EFU6UYXRKLTC6HFOPB4CRSE');

const transferTransaction = TransferTransaction.create(
    Deadline.create(),
   recipientAddress,
    [XEM.createRelative(10)],
    EmptyMessage,
    NetworkType.MIJIN_TEST);

const signedTransaction = account.sign(transferTransaction);
  1. 署名が済めば、ALICE はネットワークへ トランザクションをアナウンス できます。トランザクションがネットワークへ到着し、トランザクションオブジェクトを返却するのを待つために、TransactionHttp.announceSync の代わりに TransactionHttp.announce を使用してください。その後に ALICE は BOB にメールを送信することができます。
View Code¶
const transactionHttp = new TransactionHttp('http://0.0.0.0:9000');

transactionHttp
    .announceSync(signedTransaction)
    .subscribe(x => {
        console.log(x);
        // TODO: send email to Bob
    },
    err => {
        console.error(err);
    }
);

トランザクションが正当であれば、nem2-camel は Transaction オブジェクトを返します。トランザクションには 未確認 状態があることを強調することが重要です。ALICE またはあなたは、追加のアクションを実行する前に 幾分かの承認を待つ 必要があるかもしれません。

Catapult REST サーバーがエラーをスローした場合、購読メソッドは TransactionStatus オブジェクトを返す error 関数を呼び出します
