    • 転送トランザクションの送信¶

    モザイク とメッセージを2つのアカウント間で転送します。

    前提条件¶

    • 入門セクション を終了している
    • cat.currency を保有するアカウント

    背景¶

    転送トランザクションの送信

    Alice は Bob のアドレスである SD5DT3-CH4BLA-BL5HIM-EKP2TA-PUKF4N-Y3L5HR-IR54 に 10 cat.currency を送ろうとしています。

    トランザクションの監視¶

    アカウントがトランザクションをアナウンスすると、サーバは常に OKを返します。OK のレスポンスを受信することは、トランザクションが正当であることを意味するわけではありません。トランザクションをアナウンスする前に監視を始めておくのがよいでしょう。

    トランザクションのライフサイクルを理解するために、ターミナルを3つ開くことをおすすめします。最初のターミナルには バリデーションエラーのある アナウンスされたトランザクションの監視 。

    nem2-cli monitor status

    未承認 の監視はトランザクションがネットワークに到着したことを確認できますが、まだブロックには含まれてません。

    nem2-cli monitor unconfirmed

    トランザクションが含まれると confirmed の下に表示されるでしょう:。

    nem2-cli monitor confirmed

    それではコードを見ていきましょう¶

    1. Bob のアドレスが受信者の 10 cat.currency とした転送トランザクションを作成してください。
    View Code¶
    const recipientAddress = Address.createFromRawAddress('SD5DT3-CH4BLA-BL5HIM-EKP2TA-PUKF4N-Y3L5HR-IR54');

const transferTransaction = TransferTransaction.create(
    Deadline.create(),
    recipientAddress,
    [NetworkCurrencyMosaic.createRelative(10)],
    PlainMessage.create('Welcome To NEM'),
    NetworkType.MIJIN_TEST);
    View Code¶
    const recipientAddress = Address.createFromRawAddress('SD5DT3-CH4BLA-BL5HIM-EKP2TA-PUKF4N-Y3L5HR-IR54');

const transferTransaction = TransferTransaction.create(
    Deadline.create(),
    recipientAddress,
    [NetworkCurrencyMosaic.createRelative(10)],
    PlainMessage.create('Welcome To NEM'),
    NetworkType.MIJIN_TEST);

    すでにお気づきだと思いますが、転送トランザクションではモザイクの配列をパラメータとして必要とするので、一度で複数のモザイクを添付したトランスファートランザクションを送ることができます。

    複数のモザイクを所有している場合は、同じトランザクションでそれらを一緒に送信できます:

    View Code¶
        [new Mosaic( new MosaicId('7cdf3b117a3c40cc'), UInt64.fromUint(1000)),
        NetworkCurrencyMosaic.createRelative(10)],
    View Code¶
        [new Mosaic( new MosaicId('7cdf3b117a3c40cc'), UInt64.fromUint(1000)),
        NetworkCurrencyMosaic.createRelative(10)],

    注釈

    NEM は主に絶対量を扱います。絶対量を取得するには、送ってほしいアセットの量に 10divisibility を乗じてください。例えば、モザイクが 可分性 2 である場合、10 単位 (相対) 送信するには 1000 (絶対) と定義します。

    1. Alice のアカウントでトランザクションに署名します。

    注釈

    自分のネットワークでのみ妥当なトランザクションを作るには、最初のブロック生成ハッシュを含めます。新しいタブで http://localhost:3000/block/1 を開いて meta.generationHash の値をコピーします。

    View Code¶
    const privateKey = process.env.PRIVATE_KEY as string;
const account = Account.createFromPrivateKey(privateKey,NetworkType.MIJIN_TEST);
const networkGenerationHash = process.env.NETWORK_GENERATION_HASH as string;

const signedTransaction = account.sign(transferTransaction, networkGenerationHash);
    View Code¶
    const privateKey = process.env.PRIVATE_KEY;

const account = Account.createFromPrivateKey(privateKey,NetworkType.MIJIN_TEST);

const signedTransaction = account.sign(transferTransaction);
    1. 署名したら トランザクションをネットワークへアナウンス できます。
    View Code¶
    const transactionHttp = new TransactionHttp('http://localhost:3000');

transactionHttp
    .announce(signedTransaction)
    .subscribe(x => console.log(x), err => console.error(err));
    View Code¶
    const transactionHttp = new TransactionHttp('http://localhost:3000');

transactionHttp
    .announce(signedTransaction)
    .subscribe(x => console.log(x), err => console.error(err));
    View Code¶
     nem2-cli transaction transfer --recipient SD5DT3-CH4BLA-BL5HIM-EKP2TA-PUKF4N-Y3L5HR-IR54 --mosaics @cat.currency::10000000 --message "Welcome to NEM"
    1. アカウントのトランザクションの ステータス を監視するターミナルを開いてください。何も表示されていないことを確認してください。ターミナルにはエラーが表示された場合は エラーコード で意味を確認できます。

    未承認 を監視しているターミナルに新しいトランザクションが現れたはずです。この時点でトランザクションはネットワークに到達しましたが、ブロックに取り込まれるかどうかは不明です。

    ブロックに取り込まれた場合、トランザクションの処理が始まり、トランザクションに記載された総量が送信者アカウントから受信者アカウントへ転送されます。
