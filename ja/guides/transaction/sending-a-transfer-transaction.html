転送トランザクションの送信¶

モザイク とメッセージを２つのアカウント間で転送します。

前提条件¶

  • 入門セクション を終了していること
  • NEM2-SDK または CLI
  • テキストエディタ または IDE
  • XEM を保有したアカウント

背景¶

転送トランザクションの送信

ALICE は BOB のアドレスである SD5DT3-CH4BLA-BL5HIM-EKP2TA-PUKF4N-Y3L5HR-IR54 に 10 XEM を送ろうとしています。

トランザクションの監視

アカウントがトランザクションをアナウンスすると、サーバは常に OKを返します。OK のレスポンスを受信することは、トランザクションが正当であることを意味するわけではありません。トランザクションをアナウンスする前に監視を始めておくのがよいでしょう。

トランザクションのライフサイクルを理解するために、ターミナルを3つ開くことをおすすめします。最初のターミナルには バリデーションエラーのある アナウンスされたトランザクションの監視 。

$> nem2-cli monitor status

未承認 の監視はトランザクションがネットワークに到着したことを確認できますが、まだブロックには含まれてません。

$> nem2-cli monitor unconfirmed

トランザクションが取り込まれると confirmed の下に表示されるでしょう:。

$> nem2-cli monitor confirmed

それではコードを見ていきましょう¶

ALICE は BOB に 10 XEM 送ろうとしています。彼女は Welcome to NEM のようなメッセージを含めたいです。

  1. Bob のアドレスを受信者とし、要求された10 XEM とメッセージを追加して、トランスファートランザクションを作成してください。
View Code¶
const recipientAddress = Address.createFromRawAddress('SD5DT3-CH4BLA-BL5HIM-EKP2TA-PUKF4N-Y3L5HR-IR54');

const transferTransaction = TransferTransaction.create(
    Deadline.create(),
    recipientAddress,
    [XEM.createRelative(10)],
    PlainMessage.create('Welcome To NEM'),
    NetworkType.MIJIN_TEST);

トランザクションは作られましたが、まだネットワークへはアナウンスされていません。ネットワークがその正当性を確認できるように、ALICE が彼女のアカウントでトランザクションに署名をします。

  1. ALICE のアカウントでトランザクションに署名します。

const privateKey = process.env.PRIVATE_KEY as string;

const account = Account.createFromPrivateKey(privateKey,NetworkType.MIJIN_TEST);

const signedTransaction = account.sign(transferTransaction);

  1. 署名したら トランザクションをネットワークへアナウンス します。

  1. Open the terminal where you are monitoring account transactions status. It should be empty. If there is an error, you can check the error code here.

未承認 を監視しているターミナルに新しいトランザクションが現れたはずです。この時点でトランザクションはネットワークに到達しましたが、ブロックに取り込まれるかどうかは不明です。

ブロックに取り込まれた場合、トランザクションの処理が始まり、トランザクションに記載された総量が送信者アカウントから受信者アカウントへ転送されます。

次は？¶

同じトランザクションで複数のモザイクを送信する

複数のモザイクを追加する

同じトランザクションで複数のモザイクを送信する

すでにお気づきだと思いますが、転送トランザクションではモザイクの配列をパラメータとして必要とするので、一度で複数のモザイクを添付したトランスファートランザクションを送ることができます。

複数のモザイクを所有している場合は、同じトランザクションでそれらを一緒に送信してください:

        XEM.createRelative(10)],
