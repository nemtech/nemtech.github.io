    マルチシグトランザクションの送信¶

    マルチシグ に関与するトランザクションを送り、アグリゲートボンドトランザクション がどのように動作するのかを学びます。

    背景¶

    アグリゲートコンプリートトランザクションの送信

    Alice と Bob は一緒に住んでいて、それぞれのアカウントを持っています。また、Bob が外出するとき、彼が、自分と Alice のために食料雑貨を購入できるように、共有アカウントも持っています。

    この共有アカウントは NEM において 1-of-2 マルチシグと呼びます。これはトランザクションをブロックに取り込んでもらうのに、1人の連署者が必要なことを意味します。

    マルチシグアカウントには連署者アカウントが紐付いていること、そのアカウント自身からはトランザクションを開始できないことを留意してください。連署者だけがトランザクションを開始することができます。

    前提条件¶

    • 転送トランザクション送信ガイド を終了している
    • アカウントをマルチシグへ変換ガイド を終了している
    • アカウントの作成 方法を理解している
    • cat.currency を保有するマルチシグアカウント
    • cat.currency を保有する連署者アカウント

    それではコードを見ていきましょう¶

    Bob はバスケットをいっぱいにして、支払いをする準備が整いました。レジ係の画面には、購入費用が 10 cat.currency と表示されています。

    1. Bob は Alice と共有しているマルチシグアカウントの公開鍵と、トランザクションのアナウンスを開始するために、彼の秘密鍵を知っている必要があります。
    View Code¶
    const transactionHttp = new TransactionHttp( 'http://localhost:3000');

const cosignatoryPrivateKey = process.env.COSIGNATORY_1_PRIVATE_KEY as string;
const cosignatoryAccount = Account.createFromPrivateKey(cosignatoryPrivateKey, NetworkType.MIJIN_TEST);

const multisigAccount = PublicAccount.createFromPublicKey(multisigAccountPublicKey, NetworkType.MIJIN_TEST);

    次は？¶

    アカウントを 1-of-2 の代わりに 2-of-2 とした場合、どうなるでしょうか？

    すべての連署者の署名が揃っていない場合は、アグリゲートボンド として送信しなければなりません。

    アグリゲートボンドトランザクションの送信

    1. アグリゲートボンドトランザクションを 監視する ために新しくターミナルを開いてください。
    $> nem2-cli monitor aggregatebonded --address <your-address-here>
    1. 前のコードを変更して、トランザクション`を アグリゲートボンド として定義します。
    View Code¶
    const aggregateTransaction = AggregateTransaction.createBonded(
    Deadline.create(),
    [transferTransaction.toAggregate(multisigAccount)],
    NetworkType.MIJIN_TEST);

const signedTransaction = cosignatoryAccount.sign(aggregateTransaction);
console.log(signedTransaction.hash);
    View Code¶
    const aggregateTransaction = AggregateTransaction.createBonded(
    Deadline.create(),
    [transferTransaction.toAggregate(multisigAccount)],
    NetworkType.MIJIN_TEST);

const signedTransaction = cosignatoryAccount.sign(aggregateTransaction);
console.log(signedTransaction.hash);
    1. アグリゲートトランザクションがボンドとなった場合、Bob はネットワークのスパム防止のための 10 cat.currency でロックする必要かあります。すべての連署者がトランザクションへ署名すると、cat.currency の総量は Bob のアカウント上で再度有効になります。ハッシュロックトランザクション が承認された後に、 アグリゲートボンドトランザクションをアナウンス します。
    View Code¶
    const lockFundsTransaction = LockFundsTransaction.create(
    Deadline.create(),
    new Mosaic(
        new MosaicId('0dc67fbe1cad29e3'), // Replace with your network currency mosaic id
        UInt64.fromUint(10000000)
    ),
    UInt64.fromUint(480),
    signedTransaction,
    NetworkType.MIJIN_TEST);

const lockFundsTransactionSigned = cosignatoryAccount.sign(lockFundsTransaction);

listener.open().then(() => {

    transactionHttp
        .announce(lockFundsTransactionSigned)
        .subscribe(x => console.log(x), err => console.error(err));

    listener
        .confirmed(cosignatoryAccount.address)
        .pipe(
            filter((transaction) => transaction.transactionInfo !== undefined
                && transaction.transactionInfo.hash === lockFundsTransactionSigned.hash),
            mergeMap(ignored => transactionHttp.announceAggregateBonded(signedTransaction))
        )
        .subscribe(announcedAggregateBonded => console.log(announcedAggregateBonded),
            err => console.error(err));
});

    注釈

    クライアントサイドで使用する場合は(例, Angular, React, Vue) リスナーの実装は変わります 。

    1. Alice のアカウントで アグリゲートトランザクションに署名します 。最初のステップで出力されたトランザクションハッシュを使用してください。
    $> nem2-cli transaction cosign --hash A6A374E66B32A3D5133018EFA9CD6E3169C8EEA339F7CCBE29C47D07086E068C --profile alice
