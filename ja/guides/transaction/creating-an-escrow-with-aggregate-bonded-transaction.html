アグリゲートボンドトランザクションを使ったエスクローの作成¶

エスクローの作成によって アグリゲートボンドトランザクション について学習します。

背景¶

エスクロー とは 一次取引当事者 のための お金や書類を第三者が受け取り 、取引当事者によって合意された条件 に応じた払い出す、またはブローカーの債務者または他の人が代理して、取引の達成または終了まで 資産を保有するブローカーに立ち上げられた口座 を指します。または資産税や保険料などの債務を支払うために、借り手の名前で保有されている信託口座を言います。

完全な説明は Wikipedia を参照してください。

この例では、2つの当事者が実質的なサービスに同意すると仮定して エスクローが即時にできること を意味します。

どのように動作するのか？

  1. 買い手と売り手が規約に同意する
  2. 買い手がエスクローへ商品代を送る
  3. 売り手が商品またはサービスを買い手へ届ける
  4. 買い手は商品またはサービスを受理する
  5. エスクローは売り手へ商品代を開放する

どのように NEM に適用するのか？

NEM に関連するコンセプトへと正規化していきます:

契約上の協約
アグリゲートトランザクション
第三者がお金を受け取り、払い出す
第三者無し
主要取引先
アカウント
取引当事者が合意した条件
トランザクションに署名
ブローカーが資金を保有するために設定した口座
アカウントは不要、アグリゲートトランザクションの使用により自動的に行われます
取引の完了または終了まで
トランザクションがブロックに含まれる

前提条件¶

  • モザイクの作成ガイド を終了していること
  • アグリゲート完了トランザクションによって支払いを送信 を終了していること
  • NEM2-SDK
  • テキストエディタ または IDE

それではコードを見ていきましょう¶

マルチアセットエスクロートランザクション

要求されているアカウントとモザイクをセットアップします

この例では、ALICE とチケット販売者は以下のモザイクを交換したいと思っています。

保有者 モザイク名 総量
ALICE nem:xem 100
チケット販売者 museum:ticket 1

続行する前に、必要なネームスペースとモザイクをセットアップしてください。

モザイク交換

100 nem:xem と 1 museum:ticket を交換するために、ALICE はチケット販売者にトランザクションを送ります。

  1. 2つの転送トランザクション を作ります:
  1. ALICE からチケット販売者へ 100 nem:xem を送信します
  2. チケット販売者から ALICE へ 1 museum:ticket を送信します
  1. これらを インナートランザクション として アグリゲートトランザクション に追加します。

アグリゲートトランザクションは、ネットワークに通知する前に必要なすべての連署者が署名した場合には 完了 となります。正当な場合はブロックに含まれます。

他の参加者からの署名が必要な状態でネットワークにアナウンスされた場合、その取引はアグリゲートボンドされたものとみなされます。

const nodeUrl = 'http://localhost:3000';
const transactionHttp = new TransactionHttp(nodeUrl);
const listener = new Listener(nodeUrl);

const alicePrivateKey = process.env.PRIVATE_KEY as string;
const aliceAccount = Account.createFromPrivateKey(alicePrivateKey, NetworkType.MIJIN_TEST);

const ticketDistributorPublicKey = 'F82527075248B043994F1CAFD965F3848324C9ABFEC506BC05FBCF5DD7307C9D';
const ticketDistributorPublicAccount = PublicAccount.createFromPublicKey(ticketDistributorPublicKey, NetworkType.MIJIN_TEST);

const aliceToTicketDistributorTx = TransferTransaction.create(
    Deadline.create(),
    ticketDistributorPublicAccount.address,
    [XEM.createRelative(100)],
    PlainMessage.create('send 100 nem:xem to distributor'),
    NetworkType.MIJIN_TEST);

const ticketDistributorToAliceTx = TransferTransaction.create(
    Deadline.create(),
    aliceAccount.address,
    [new Mosaic(new MosaicId('museum:ticket'), UInt64.fromUint(1))],
    PlainMessage.create('send 1 museum:ticket to alice'),
    NetworkType.MIJIN_TEST);

  1. アグリゲートトランザクションがボンドであるとき、ALICE は最低 10 XEM でロックする必要があります。チケット販売者がアグリゲートトランザクションに署名すると、ロックされた XEM の総量が ALICE のアカウントで再び利用可能になり、交換が完了します。

const aggregateTransaction = AggregateTransaction.createBonded(Deadline.create(),
    [aliceToTicketDistributorTx.toAggregate(aliceAccount.publicAccount),
        ticketDistributorToAliceTx.toAggregate(ticketDistributorPublicAccount)],
    NetworkType.MIJIN_TEST);

const signedTransaction = aliceAccount.sign(aggregateTransaction);

const lockFundsTransaction = LockFundsTransaction.create(
    Deadline.create(),
    XEM.createRelative(10),
    UInt64.fromUint(480),
    signedTransaction,
    NetworkType.MIJIN_TEST);

const lockFundsTransactionSigned = aliceAccount.sign(lockFundsTransaction);

listener.open().then(() => {

    transactionHttp
        .announce(lockFundsTransactionSigned)
        .subscribe(x => console.log(x), err => console.error(err));

    listener
        .confirmed(aliceAccount.address)
        .pipe(
            filter((transaction) => transaction.transactionInfo !== undefined
                && transaction.transactionInfo.hash === lockFundsTransactionSigned.hash),
            mergeMap(ignored => transactionHttp.announceAggregateBonded(signedTransaction))
        )
        .subscribe(announcedAggregateBonded => console.log(announcedAggregateBonded),
            err => console.error(err));
});

注釈

クライアントサイドで使用する場合は(例, Angular, React, Vue) リスナーの実装は変わります 。

この取引をアグリゲートトランザクション無しに可能でしょうか？¶

次のリストのようなことが起こりうるので 安全ではありません 。

  • 買い手は支払いませんでした。
  • 売り手は実質的な商品を送りませんでした。

次は？¶

販売者はまだアグリゲートボンドトランザクションに署名していないため、交換は完了していません。アナウンスされたアグリゲートボンドトランザクションに署名をするガイド を読むことを検討してください。

その後は、複数の参加者間でモザイクの交換を試してみてください。
