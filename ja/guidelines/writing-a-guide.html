    ガイドを記述する¶

    新しいガイドの記述を検討してくれてありがとう！

    始める前に¶

    1. nem2-docs リポジトリ issues を開き、記述しようとしているアイディアを探してください。それらは 「great new issue」 ラベルを持っています。
    2. コメントを追加してその1人に貢献するか、あなたのガイドのアイデアに新しい issue を作成するかを選択します。
    3. 次のカテゴリのいずれかにガイドを分類してください:
    • アカウント
    • アカウントフィルタ
    • アグリゲートトランザクション
    • ブロック
    • クロスチェーンスワップ
    • ハーベスティング
    • モザイク
    • ネームスペース
    • ノード
    • レシート
    • 転送トランザクション
    • ツール

    注釈

    GitHub issue を作成することで新しいカテゴリを提案できます。

    ガイドを記述する¶

    1. https://help.github.com/articles/fork-a-repo/ をフォークして`nem2-docs <https://github.com/nemtech/nem2-docs>`_ リポジトリをクローンしてください。
    $> git clone <url>
    1. Python 2.7 または 3.4+ と pip がインストールされていることを確認してください。
    $> python --version
    1. pipを使用した要求環境のインストール
    $> pip install -r requirements.txt
    1. ガイドのフォルダ内に新しく rst を作成します。
    $> mkdir source/guides/<folder_name>/<title>.rst
    1. あなたのコンテンツを整理するためにこのテンプレートに従ってください。
    :orphan:

.. post:: 18 Aug, 2018
    :category: <category>
    :excerpt: 1
    :nocomments:
    :author: <your_name_or_username>

#####
Title
#####

Objective after finishing the guide.

**********
Background
**********

Explain necessary concepts before starting to code.

*************
Prerequisites
*************

- A
- B
- C

************************
Let’s get into some code
************************

Present the code and step-by-step explanation.

************
What's next?
************

Is there any extra exercise that readers could try on their own?
    1. あなたのコードとガイドを書いてください！ 書式は restructured text チートシート を確認してください。
    2. コード例 を``source/resources/examples/<language_or_tool>`` の下に追加してください。rst ファイルの中からコード断片をレンダリングすることもできます。
    .. example-code::

    .. literalinclude:: <relative_url>.ts
        :language: typescript
        :lines: 20-40

    1. あなたの変更点をテストしプレビューしてください。

      $> make livehtml

    2. 変更をプッシュして プルリクエスト をしてください。リポジトリのメンテナがドキュメント記述のスタイルに沿ったコンテンツの構成と編集を行います。
