コアシステムスキーマ¶

注釈

The catbuffer schemas repository defines how each transaction type should be serialized. In combination with the catbuffer-generators project, developers can generate builder classes for a given set of programming languages.

投票リンクキー¶

VotingKeyLinkTransaction¶

このトランザクションは ファイナライズ済み ブロックに投票するノードオペレーターに必要です。

投票キーを関連付けるために、一定期間のアカウントを持つ公開鍵で、 VotingKeyLinkTransaction をアナウンスします。アカウントは同時に最大 3 つの異なる投票キーにリンクできます。

本番環境の推奨設定では、最初のキーの有効期限が切れた後にキーが確実に登録されるように、常に異なる endPoint 値を持つ、最低でも 2 つのキーをリンクします。詳細は ガイド を参照してください。

Version: 0x01

EntityType: 0x4143

Inlines:

  • Transaction または EmbeddedTransaction
プロパティ Type Description
linkedPublicKey VotingKey リンクした公開鍵
startPoint FinalizationPoint ファイナライゼーション開始ポイント
endPoint FinalizationPoint ファイナライズ終了ポイント。値は startPoint + minVotingKeyLifetime より大きく、 startPoint + maxVotingKeyLifetime より小さくする必要があります。ライフタイムプロパティは ネットワークごとに構成可能 です。
linkAction LinkAction アカウントリンクアクション

Vrf キーリンク¶

VrfKeyLinkTransaction¶

このトランザクションはすべての ハーベスティング適格アカウント に必要です。

VRF 公開鍵をアカウントにリンクするために VrfKeyLinkTransaction をアナウンスします。キーはブロック生成とリーダー/参加者の選択をランダム化するために使用されます。

Version: 0x01

EntityType: 0x4243

Inlines:

  • Transaction または EmbeddedTransaction
プロパティ Type Description
linkedPublicKey Key リンクした公開鍵
linkAction LinkAction アカウントリンクアクション
