報酬プログラム¶

ノード はブロックを生成しては保存するので、 いかなるブロックチェーンにおいても大黒柱です 。したがって、ネットワーク内には常にできるだけ多くのノードがアクティブであることが 重要 です。

ノードがブロックを生成すると取得する ブロック報酬 (手数料 と インフレーション) による直接な報酬の他に、 Symbol ユーザによるノードの作成および維持を奨励することを目的とした 一連の特別な報酬プログラムを提供 します。

これらの各プログラムは ネットワークのさまざまな側面 で報酬を与えます:

  • スーパーノードプログラム: 強力な、高可用、高スループットのノード向け
  • 投票ノードプログラム: ファイナライズと Symbol のフォーク解決メカニズムを支持するノード向け

ほとんどの場合、これらのプログラムの恩恵を受けたいノード所有者は、明示的に 登録 する必要があります。各プログラムへの登録方法は、以下の説明に ガイド が含まれています。

警告

None of the reward programs has launched yet. The information shown below is subject to change until the programs officially start.

スーパーノードプログラム¶

このプログラムの目的は、ネットワークの円滑な機能を保証するために、十分な数の 強力なノード が常に利用可能であることを保証するためです。

要求事項¶

スーパーノードとは、次の追加のハードウェアおよび運用要件を満たす、通常ノードです:

パラメータ 要求事項
ノードタイプ デュアルノード (Peer と API ノードの両方) である必要があります。
帯域 5’000 kbit/s 以上、ネットワーク上にある、他の3つのランダムなノードから、2MB のファイルダウンロードによって計測されます。
チェーン高 チェーンは基準の高さから 4 ブロック 以内に同期できていなければなりません。
チェーン部分 チェーンの 50個のランダムなブロックハッシュ を正確にアップロードします。これらを正確さのために参照チェーンのハッシュと比較します。
計算力 最低でも 1 秒あたり 2,000回反復 をハッシュ計算を行います。ノードは 32 byte シードを 10,000 回の反復の計算を要求されます。速度と精度を測定します。
Bonded Deposit ノードには最低でも 1, 2, 3 百万 symbol.xym の保有が求められます。総量で報酬形態が変わります。 (以下を参照)
バージョン Symbol must be up to date. Nodes should be updated within 10 days of each new Symbol release.
Ping 200ms 以下のレスポンス ノードはネットワーク上の他のノードに 20 回のランダムな ping の実行を要求され、最低 1 回の ping 応答がテストに合格しなければなりません (各ノードは 5 回 ping され、応答時間は平均化されます)
応答性 1秒: 10 個のブロックチェーン高がリクエストされ、最低でも 9 個を 1 秒以内に応答しなければなりません。

To ensure all of the above, a lightweight monitoring agent is installed on the node which reports to a controller on the network. The controller performs all the above checks a few times per day on each registered supernode and makes daily payments.

報酬¶

Total daily rewards

Year 1 15’000 symbol.xym
Year 2 15’000 symbol.xym
Year 3 15’000 symbol.xym
Year 4 14’000 symbol.xym
Year 5 10’000 symbol.xym
Year 6 5’000 symbol.xym

This program will last for 6 years after the launch date of Symbol, with decreasing rewards each year.

Supernodes are divided in 3 tiers according to their staked symbol.xym: More than 1 million, more than 2 million and more than 3 million.

The available Total daily rewards are split among all program participants according to their tier in the following fashion:

  • Each tier 1 nodes gets 17.33% of the Total daily reward divided by the total number of participating nodes.
  • Each tier 2 nodes gets 46.67% of the Total daily reward divided by the total number of participating nodes.
  • Tier 3 nodes get the remainder, uniformly distributed among them.

完全な 利用規約 はこちらを参照してください。

注釈

スーパーノードプログラムに参加するにはこのガイドに従ってください

投票ノードプログラム¶

Symbol provides fast response times even when network disconnections isolate some of the nodes. When connectivity is restored any conflicts which might have arisen (network forks) are resolved by the ファイナライゼーション process. This process requires some nodes to periodically vote on which branches are valid according to their records. Blocks deemed invalid are then rolled back.

This program aims at ensuring that enough voting nodes are always present in the network so fork resolution is conducted reliably.

要求事項¶

投票ノードとは、次の追加要件を満たす、通常ノードです:

パラメータ 要求事項
Bonded Deposit ノードには最低でも 300 万 symbol.xym のステークが必要です。
投票キー The node must signal its interest in becoming a voting node by registering its Voting Key.

Although not mandatory, it is recommended that voting nodes meet the same hardware requirements as supernodes. In fact, it is expected that participants of the Voting Node Program will also enroll to the Supernode program.

報酬¶

5% of all block harvesting rewards is set aside for voting nodes. After each finalization round these rewards are evenly distributed among all nodes that participated in that round (and paid on a daily basis). There is no explicit enrollment required for this program.

注釈

Symbol Bootstrap を使用してノードを作成した場合は このガイドに従って投票 を有効にしてください。

または このガイドに従い、手動で投票を有効化 します。

完全な 利用規約 はこちらを参照してください。
