    REST API¶

    Catapult REST API は HTTP と WebSockets を NEM ブロックチェーン上で読み書きアクションが実行できるように併合します。

    HTTP リクエスト¶

    Catapult REST はポート 3000 番を使用します。HTTP GET 、 PUT そして POST リクエストを受け付けます。

    Catapult REST がローカルで動いていると仮定すると、HTTP GET リクエストはブラウザからこのように実行できます:

    http://localhost:3000/<path-to-API-request>

    HTTP PUT と POST リクエストはボディ部にJSON形式を使用します。リクエストはデータをJSON形式で返します。この種のリクエストはそれが可能なプラグインを利用しないことにはブラウザから実行することができません。

    ツール¶

    NEM2-SDK

    NEM2 ソフトウェア開発キット とは NEM2 コンポーネント、このような追加ツール、ライブラリ、アプリケーションなどを作成するための主要なソフトウェア開発ツールです。

    注釈

    API に直接呼び出す代わりに NEM2-SDK の利用を考慮してください。

    • リファレンス
    • ガイド

    Insomnia

    Mac, Windows, Linux 向けのオープンソースの HTTP クライアント。

    1. あなたのOS用の Insomnia app をダウンロードしてください。
    2. NEM 用の Insomnia 仕様 をインポートしてください。

    シリアル化¶

    Catbuffer ライブラリ は Catapult のエンティティをシリアライズ/デシリアライズするプロトコルを定義します。SDK とアプリケーションは REST トランザクションエンドポイントと疎通するために生成されたコードを利用します。

    NEM2-SDK シリアル化モジュール

    このライブラリは以下の特性を達成します:

    効率的なメモリ使用

    大きなネットワークはたくさんのトランザクションを計算します。バイナリに最適化された仕事はコミュニケーションを高速にします。さらに、メモリバッファからのエンティティ -またはそれらの一部- の読み込みはメモリを効率的に使用します。

    柔軟性

    REST トランザクションエンドポイント はブロックチェーンの状態を更新するための呼び出しをハンドリングします。シリアライズされたトランザクションのペイロードは POST 呼び出しのボディに追加されます。これらのエンドポイントにより API の規約を変更せずに、サーバ側に新しい機能を追加することができます。

    再利用性

    アプリケーションは依存性の管理することなく、生成されたコードを組み込むことができます。これは安全性の高い環境において、特に望ましいです。さらに共通のコードベースを共有することは、より少ない労力で新しい機能を追加することを可能にします。

    スキーマ はエンティティのデータ構造を定義しています。このライブラリは定義済みエンティティを、シリアライズまたはデシリアライズするために必要な最低限のコードを生成します。

    利用可能な言語の1つで決定されたスキーマのコードを生成してください。たとえば、転送トランザクションの C ++ 用トランザクションビルダーを生成するには、次のコマンドを実行します。

    $> python main.py --schema schemas/transfer/transfer.cats --generator cpp_builder

    ジェネレータは _generated/cpp_builder フォルダ以下に新しいファイルを生成します。必要に応じて、異なる入力スキーマ (-s) またはジェネレータ (-g) を使用して、このプロセスを繰り返してください。

    HTTP エラー¶

    ステータスコード 説明
    200 OK リクエストは成功しました。
    202 Accepted リクエストは処理のために受理されましたが、処理は完了していません。
    400 Bad request リクエストのシンタックスを確認してください。
    404 Not Found リソースは存在しません。
    409 Conflict 引数を確認してください。
    500 Internal error 予期しない状態です。

    HTTP ステータス¶

    キー 説明
    コード キャメルケースによるエラー識別子
    メッセージ 人間が読めるフォーマットによるエラー説明

    例

    {
  "code": "InvalidArgument",
  "message": "accountId has an invalid format"
}

    uint64: lower と higher¶

    Javascript は 32 bit の値を扱います。64 bit の表現を可能にするために、API は lower と higher の２つの部分にエンコードされた数値を返します。

    lowerとhigherを単一の数値へと変換する方法 を確認してください。

    WebSockets¶

    WebSockets はブロックチェーン上でトランザクションやイベントが発生した通知を受け取ることを可能にします。この通知は API の返却を待つことなくリアルタイムで行われます。

    NEM2-SDK で API WebSocket と疎通するには Listeners を使用します。

    チャンネル¶

    ブロック

    ブロックチャンネルは新しいブロックごとに通知します。メッセージにはブロックの情報が含まれます。

    confirmedAdded/<ADDRESS>

    承認済追加チャンネルは、アドレスに関連するトランザクションがブロックに含まれたときに通知します。メッセージにはトランザクションが含まれます。

    unconfirmedAdded/<ADDRESS>

    未承認追加チャンネルは、アドレスに関連するトランザクションが未承認状態にあり、ブロックに含まれるのを待っているときに通知します。メッセージにはトランザクションが含まれます。

    このチャンネルからの通知が届くシナリオとしては、トランザクションが PUT /transaction HTTP エンドポイントによりアナウンスされるか、アグリゲートボンドトランザクションのすべての署名が揃い partial から unconfirmed へ変わるときです。

    unconfirmedRemoved/<ADDRESS>

    未承認削除チャンネルはアドレスに関連するトランザクションが未承認状態であったが、そうでなくなったときに通知を行います。メッセージにはトランザクションハッシュが含まれます。

    このチャンネルが通知するシナリオとしては、トランザクションが承認済になった、または期限を迎えてブロックに含まれなくなったときです。

    partialAdded/<ADDRESS>

    パーシャル追加チャンネルはアドレスに関連するアグリゲートボンドトランザクションが partial な状態にあり、すべての 必要な連署者 を待機しているときに通知を行います。メッセージにはトランザクションハッシュが含まれます。

    このチャンネルが通知するシナリオとしては、アグリゲートボンドトランザクションが PUT /transaction/partial 経由でネットワークへアナウンスされたときです。

    partialRemoved/<ADDRESS>

    パーシャル削除チャンネルはアドレスに関連するトランザクションがパーシャルな状態であったが、そうでなくなったときに通知を行います。メッセージにはトランザクションハッシュが含まれます。

    このチャンネルが通知するシナリオとしては、トランザクションが未承認になった、または期限を迎えてブロックに取り込まれなくなったときです。

    cosignature/<ADDRESS>

    連署チャネルはアドレスに関連する 連署署名トランザクション が、パーシャル状態のアグリゲートボンドトランザクションに追加されたときに通知します。メッセージには連署したトランザクションが含まれます。

    status/<ADDRESS>

    ステータスチャンネルはアドレスに関連するトランザクションでエラーが発生したときに通知を行います。メッセージにはエラーとトランザクションハッシュが含まれます。

    Status errors¶

    このセクションでは、トランザクションをアナウンスした後に、ステータスチャンネルが返却するエラーメッセージの説明をします。

    設定パラメータは 編集可能 です。公開ネットワークでは異なる可能性があります。

    Neutral バリデーション結果は成功でも失敗でもありません。

    Failure_Core_Insufficient_Balance アカウントの残高が不十分なため、バリデーションに失敗しました。
    Failure_Core_Too_Many_Transactions ブロックにトランザクションが多すぎるため、バリデーションに失敗しました。
    Failure_Core_Nemesis_Account_Signed_After_Nemesis_Block ネメシスブロックの後にエンティティがネメシスアカウントから生成されたため、バリデーションに失敗しました。
    Failure_Core_Wrong_Network エンティティに間違ったネットワークが指定されているため、バリデーションに失敗しました。

    Failure_Hash_Exists エンティティのハッシュがすでに存在するため、バリデーションに失敗しました。
    Failure_Signature_Not_Verifiable 署名の検証に失敗したため、バリデーションに失敗しました。
