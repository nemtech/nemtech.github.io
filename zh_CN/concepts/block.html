    区块¶

    每个加密货币的核心要素是一个称为区块链的公共账本。它有许多区块组成。

    由于链中的区块是有序的，因此区块链中保存了完整的交易历史记录。后续添加的区块使区块链中的的高度加一。

    区块作为永久介质存储在数据库中。 NEM称链中的首区块为nemesis 块。

    NEM每 15 秒生成一个区块。这可迅速确认交易，方便日常交易。

    注解

    配置参数是 可更改的。公共网络配置可能有所不同。

    区块的创建¶

    区块由 账户 创建。创建新区块的过程称为 收获。 收获账户 - 称为收获者 -他们会获得区块中的交易费。这使收获者有动力尽可能将多些交易添加到区块中。

    模式¶

    BlockHeader¶

    内联：

    • VerifiableEntity
    • EntityBody
    属性 类型 说明
    height uint64 区块链的高度。 每个区块链都有各自的高度。区块与随后的区块高度相差1。
    timestamp uint64 自创建nemesis 块以来经过的秒数。
    difficulty uint64 平均难度除以最后 60 个区块的平均创建时间。如果新的难度比前一个区块的难度高或低超过5%，那么难度将不会改变超过5%。除此之外，难度会保持在一定范围内。如果新的难度大于1015 或小于 1013，则将新的难度限制在边界上。
    feeMultiplier uint32 用于区块中的交易的费用乘数。
    previousBlockHash 32 字节 (二进制) 前一个区块的哈希值。
    blockTransactionHash 32 字节 (二进制) 块中包含的交易形成哈希并组成 merkle tree。树的根为总结。
    blockReceiptsHash 32 字节 (二进制) The collection of receipts are hashed into a merkle tree and linked to a block. The block header stores the root hash.
    stateHash 32 字节 (二进制) 对于每一个区块，区块链的状态将被存储在RocksDB中，形成 patricia tree 。 树的根总结了该区块的区块链的状态。
    beneficiaryPublicKey 32 字节 (二进制) 由收割者指定的非必须受益人的公钥。

    Version: 较高的字节代表网络识别码。

    Id 说明
    0x68 (MAIN_NET) 公共主要网络
    0x98 (TEST_NET) 公共测试网络
    0x60 (MIJIN) 私有网络
    0x90 (MIJIN_TEST) 私有测试网络

    Type: 区块的类型。

    Id 说明
    0x8043 Nemesis块
    0x8143 区块

    VerifiableEntity¶

    属性 类型 说明
    signature 64 字节 (二进制) 签名者生成的实体签名。

    EntityBody¶

    属性 类型 说明
    signer 32 字节 (二进制) 实体签名者的 公钥。
    version uint16 结构的版本。
    type uint16 实体类型。 有关交易类型，请参阅 交易类型.
