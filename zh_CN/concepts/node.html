    节点¶

    NEM区块链平台是由节点网络构建成的。这些节点提供了一个功能强大，稳定且安全的平台。这个平台可以运行、搜索、并把智能资产交易以不变式记录到区块链总账中。

    Catapult（投石车）的性能优势：四层架构

    这种四层体系结构允许开发人员在不中断其他层的情况下更新任何一层，从而提高了安全性。

    P2P组件¶

    Repository: Catapult Server

    P2P节点构成了区块链的主干，它们让网络更扎实，因为无法通过消除单个实体而使整个网络停止工作。节点的作用是 验证交易 和 区块，运行共识演算法，创建新块，并通过网络传播更改。

    API把新交易推送到P2P网络，通过网络 被收录在区块里 或被丢弃。区块被处理完毕后：

    • 区块的二进制文件以Flat File（平面文件）方式保存在磁盘上。
    • 已更新的链状态保存在内存或RocksDB（可配置）中。

    节点信誉¶

    公共网络允许所有人运行节点。但某些节点可能共享无效信息甚至试图干扰网络。

    为了减少通信企图，节点会跟踪之前通信的结果。

    当一个节点连接到远程节点时，它首先会对该远程节点增加信任。反之，会减低对其的信任计数器减分。同样，节点在处理所请求的数据之后相应地更新信任计数器。

    在这些交互中，节点为每个联系的节点分配500到10000之间的权重。

    选择远程节点的概率线性取决于该远程节点权重。 每四轮节点选择后，会变动标准来预防 Sybil attacks。 该节点选择具有高权重的节点。

    RocksDB¶

    RocksDB 存储链的状态。缓存的数据结构被序列化并作为值存储到相应的密钥中。例如，列把公钥映射到地址；把帐户状态项作为对应地址密钥值。

    将状态存储在内存中通常比使用RocksDB更快。但在RocksDB中存储状态信息需要更少的网络节点内存。

    注解

    在拥有大量账户的网络中保存状态信息是很方便的。

    API组件¶

    Repository: Catapult REST

    API的主要职责是以可读的形式存储数据。每个API实例都维持一个MongoDB，且可选择使用RocksDB来保存状态。

    这一层 验证 从REST组件接收到交易。此外，API通过ZMQ以二进制方式显示错误。

    该组件还负责收集 保税聚合交易，的共同签名。这些交易只有在完成后才会被推送到P2P节点。

    API组件在可以连接到多个P2P节点，但至少需连接到一个节点。

    MongoDB¶

    MongoDB 存储区块，交易和链状态以实现高效查询性能。

    API在以下情况更新MongoDB：

    • 一个新区块/一堆区块被完成处理后。
    • 新的未确认交易被完成处理后。

    注解

    MongoDB不应该被外部接驳。

    ZMQ¶

    ZeroMQ 是一个支持实时订阅的异步信息库。它把通知从API服务器传输到Catapult REST组件侦听的ZMQ端点。它是REST WebSockets的替代方案，旨在性能至关重要时使用。

    REST 组件¶

    Repository: Catapult REST

    REST组件处理 JSON API 客户端请求。这实现了从MongoDB读取，格式化响应，并将其返回给客户端。此组件还负责使用 WebSockets 将事件返回给客户端。

    每个REST组件连接到一个API实例，使用sockets（接口）发送新交易。
