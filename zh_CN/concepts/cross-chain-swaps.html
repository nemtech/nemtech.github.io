    跨链交换¶

    跨链交换让在不使用中介的情况下跨越 不同的区块链 进行 代币交易（例如货币交易服务）。

    公链和私链之间的原子跨链交换

    特定的交易类型，通常称为 Hashed TimeLock Contract (HTLC) 给交易所一个无需信任的环境。 哈希锁 hashlocks 和 * 时间锁 timelocks* 是这类交易的表征。您可以在 Bitcoin Wiki 上找到详尽的解释。

    换句话说，为了降低交易对象风险，支付接收方需要提供交易执行证明。如果没这样做，即使只有一个参与者未同意，锁定的资金也将在截止日期之后被释放。下图说明了跨链交换协议。

    原子跨链交换序列图

    在NEM中谈论代币时，我们实际上指的是 马赛克 。Catapult 通过 秘密锁 / 秘密证明交易 机制进行原子交换。

    SecretLockTransaction¶

    以秘密锁交易来启动跨链交换：

    1. 确定要传输到指定帐户的马赛克单位。
    2. 随机生成一组称为 证明 proof 的字节。
    3. 用其中一种演算法对获得的证据进行哈希处理以生成 秘密 secret 。
    4. 选择锁定马赛克的时间并宣布交易。

    指定的马赛克将被锁定，直到有效的 秘密证明交易 将它们解锁。

    如果在交易期限内没有完成解锁，那么锁定的金额将会返回到秘密锁交易的发起者账户。

    Version: 0x01

    实体类型: 0x4152

    内联:

    • Transaction or EmbeddedTransaction
    属性 类型 说明
    mosaic Mosaic 锁定的马赛克。
    duration uint64 锁定期限。 如果期限已达，马赛克将返回到发起者的账户。
    hashAlgorithm LockHashAlgorithm 用于处理证明的哈希演算法。
    secret 64 字节 (二进制) 经过哈希处理的证明。
    recipient 25 bytes (binary) 一旦解锁就会收到资金的地址。

    SecretProofTransaction¶

    以秘密证明交易来解锁 秘密锁交易。

    交易必须证明它知道用来解锁马赛克的 证明 。

    Version: 0x01

    实体类型: 0x4252

    内联:

    • Transaction or EmbeddedTransaction
    属性 类型 说明
    hashAlgorithm LockHashAlgorithm 用于处理证明的哈希演算法。
    secret 64 字节 (二进制) 经过哈希处理的证明。
    proofSize uint16 证明大小以字节为单位。
    proof array(byte, proofSize) 原本的证明。

    哈希锁演算法 （LockHashAlgorithm）¶

    例举: uint8

    Id 说明
    0 (Op_Sha3_256) 输入值将以sha3 256进行哈希处理。
    1 (Op_Keccak_256) 输入值以Keccak (ETH兼容）形成哈希。
    2 (Op_Hash_160) 输入值被哈希处理两次：首先是Sha-256，然后是RIPEMD-160（比特币的OP_HASH160）。
    3 (Op_Hash_256) 使用Sha-256（比特币的OP_HASH256）对输入值进行两次哈希处理。
