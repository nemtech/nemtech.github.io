    REST API¶

    Catapult REST API 结合HTTP和WebSockets在NEM区块链中执行读写操作。

    Http 请求¶

    Catapult REST 使用 3000 端口。它接受 GET, PUT 和 POST 请求。

    假设Catapult REST在本地运行，HTTP GET请求可以从浏览器执行，其形式如下:

    http://localhost:3000/<path-to-API-request>

    HTTP PUT和POST请求在请求体中使用JSON结构。请求使用JSON结构返回数据。这类请求通常不能在浏览器中执行，除非您使用一个插件使您能够执行它。

    工具¶

    NEM2-SDK

    NEM2-SDK 是创建NEM2组件(如其他工具、库或应用程序)的主要软件开发工具。

    注解

    请考虑使用 NEM2-SDK 而不是直接调用API。

    序列化¶

    Catbuffer library 定义了序列化和反序列化Catapult实体的协议。该库提供了适用于不同语言的代码生成器。SDK和应用程序将使用生成的代码与REST交易端点实现交互。

    NEM2-SDK序列化模块

    该库实现以下属性:

    记忆效率 （Memory Efficiency)

    大型网络计算大量交易。使用大小合适的二进制设备可以使通信更快。此外，从内存缓冲区读取实体 - 或者只是实体的一部分 - 能充分利用好内存。

    灵活性（Flexibility）

    REST的 交易端点（transaction endpoints) 掌控着区块链状态更新的调用。交易的序列化酬载量被附加到了POST调用的主体中。这些端点支持在不修改API协定的情况下向服务器端添加新功能。

    复用性（Reusability）

    应用程序可以在无需管理依赖关系的状况下嵌入生成的代码。这非常适用于安全性高的环境。此外，共享通用代码库可以用更少的工作添加新功能。

    模式定义（schemas define） 实体数据结构。该库生成序列化和反序列化定义实体所需的最精简代码。

    使用其中一种可用语言生成已确定模式的代码。例如, 运行以下命令以生成 c++ 转让交易的交易构建器:

    $> python main.py --schema schemas/transfer/transfer.cats --generator cpp_builder

    生成器在 _generated/cpp_builder 文件夹下创建一个新文件。根据实际情况, 请使用不同的输入架构 (-s) 或生成器 (-g) 重复此过程。

    http错误¶

    状态代码 描述
    200 ok. 请求已经成功。
    202 接受。该请求已被接受进行处理，但尚未完成处理。
    400 错误请求。检查请求语法。
    404 未找到。 资源不存在。
    409 冲突。检查你的参数。
    500 内部错误。意料外情况。

    uint64:低值和高值¶

    Javascript以32位值进行操作。为了支持高达64位值，API返回编码为 低值 和 高值 两部分的数字。

    Check how to compact lower and higher into a single value.

    WebSockets¶

    WebSockets使得接收在区块链中发生的交易或事件的通知变得可能。实时接收通知，无需轮询API等待回复。

    与API WebSockets的交互 NEM2-SDK 是用 监听器 完成的。

    通道¶

    区块

    区块通道通知每个新区块的生成。消息包含区块信息。

    confirmedAdded/<ADDRESS>

    当相关地址的交易被收录在区块内，confirmedAdded通道发出通知。 该通知包含该交易。

    unconfirmedAdded/<ADDRESS>

    当相关地址的交易还在未确认并等待被收录于区块内，confirmedAdded通道发出通知。 该通知包含该交易。

    该通道会在以下可能的情况中发出通知：通过 PUT /transaction HTTP端点向网络宣布的交易或聚合保税交易已得到所有的共签，并将其状态从 partial 更改为 unconfirmed。

    unconfirmedRemoved/<ADDRESS>

    当交易不再处于未确认状态时，unconfirmedRemoved将发出通知。该通知包含该交易哈希。

    该通道会在以下可能的情况中发出通知：交易已被确认，或者已经到达了最后期限，但没有收录在区块中。

    partialAdded/<ADDRESS>

    partialAdded通道在相关地址的聚合保税交易处于*partial*状态并等待凑齐所有*共签*时发出通知。该通知包含一个交易。

    此通道会在以下可能的情况中发出通知：通过 PUT /transaction/partial HTTP端点向网络宣布聚合保税交易。

    partialRemoved/<ADDRESS>

    partialremove通道在相关地址的交易不再处于partial状态时发出通知。该通知包含交易哈希。

    该通道会在以下可能的情况中发出通知： 交易现在处于未确认状态，或者已达到截止日期，但还未被收录在区块中。

    cosignature/<ADDRESS>

    当相关地址的*共签签名交易*添加到partial状态的聚合保税交易，cosignature通道将发出通知。该通知包含有共签名签名交易。

    status/<ADDRESS>

    状态通道在相关地址的交易出现错误时发出通知。该通知包含错误消息和交易哈希。

    Failure_Property_Signer_Address_Interaction_Not_Allowed 验证失败 因为签名者不允许与涉及交易的地址进行交互。
    Failure_Property_Mosaic_Transfer_Not_Allowed 验证失败 因为收件人禁止马赛克的转移。
    Failure_Property_Transaction_Type_Not_Allowed 验证失败 因为签名者不被允许启动交易类型。
    Failure_Transfer_Message_Too_Large 验证失败 因为信息太大。它超过了 1024 字节数限制。
    Failure_Transfer_Out_Of_Order_Mosaics 验证失败 因为马赛克无序。转让交易上的马赛克应该按ID值排序。
    Failure_Chain_Unlinked 验证失败 因为接收到的区块没有链接到现有链。
    Failure_Chain_Block_Not_Hit 验证失败 因为接收到的区块不是命中的。
    Failure_Chain_Block_Inconsistent_State_Hash 验证失败 因为接收到的区块状态哈希不一致。
    Failure_Chain_Block_Inconsistent_Receipts_Hash 验证失败 因为接收到的区块的收哈希列不一致。
    Failure_Chain_Unconfirmed_Cache_Too_Full 验证失败 因为未确认的缓存太满。
    Failure_Consumer_Empty_Input 验证失败 因为使用者的输入为空。
    Failure_Consumer_Block_Transactions_Hash_Mismatch 验证失败 因为区块交易哈希与计算值不匹配。
    Failure_Consumer_Hash_In_Recency_Cache 验证失败 因为在最近缓存中存在实体哈希。
    Failure_Consumer_Remote_Chain_Too_Many_Blocks 验证失败 因为链部分有太多区块。
    Failure_Consumer_Remote_Chain_Improper_Link 验证失败 因为链的内部链接不正确。
    Failure_Consumer_Remote_Chain_Duplicate_Transactions 验证失败 因为链部分包含重复交易。
    Failure_Consumer_Remote_Chain_Unlinked 验证失败 因为链部件没有连接到当前链。
    Failure_Consumer_Remote_Chain_Mismatched_Difficulties 验证失败 因为远程链的难度与计算的难度不一样。
    Failure_Consumer_Remote_Chain_Score_Not_Better 验证失败 因为远程链的得分并不高。
    Failure_Consumer_Remote_Chain_Too_Far_Behind 验证失败 因为远程链太落后了。
    Failure_Consumer_Remote_Chain_Too_Far_In_Future 验证失败 因为远程链时间戳太遥远。
    Failure_Extension_Partial_Transaction_Cache_Prune 验证失败 因为部分交易从临时缓存中被删除。
    Failure_Extension_Partial_Transaction_Dependency_Removed 验证失败 因为部分交易因其依赖关系从临时缓存中被删除。
